# EXPECTUM STEWARDSHIP BACKLOG 1.0

## Purpose

This backlog collects mature observations that have already become visible through Expectum’s audits, decisions, explorations and governance documents, but have not yet been technically embodied.

These are not random TODOs. They are not a new feature roadmap, a redesign brief or a new audit. They are already-seen signals that need careful stewardship before any future implementation.

The purpose of this document is to keep those observations visible without forcing premature implementation.

## Core Compass

“Does this help a participant reach recognition from which Connection may become possible?”

This question should guide future reduction, wording, navigation, visual, memory and encounter decisions.

If an element helps recognition become possible, it may deserve protection or careful embodiment.

If an element does not help recognition become possible, or hides the encounter behind noise, it should be questioned.

If two elements serve the same purpose, the quieter and clearer one should usually remain.

## Already Seen, Not Yet Embodied

### Encounter and One Meeting

- Move toward one encounter instead of three explicit destinations.
  - Status: Needs decision before implementation.
  - Stewardship note: One Meeting Decision 1.0 names the direction, but implementation must wait for a clear transition gate.

- Preserve Kohtumine, Mõttekohtumine and Avardamine as movements, not separate destinations.
  - Status: Needs decision before implementation.
  - Stewardship note: The distinction should not be erased; it should become more organic if the system moves toward one encounter.

- Restore short, quiet, proportional Aim responses where needed.
  - Status: Needs decision.
  - Stewardship note: This does not mean flattening Aim into one uniform voice. It means protecting proportion, silence and encounter from unnecessary expansion.

- Prevent Avardamine from becoming the default long answer.
  - Status: Needs decision.
  - Stewardship note: Expansion is valuable when it serves the meeting, but it should not become the assumed highest form of response.

- Reconsider “Jätka kohtumist”.
  - Status: Needs decision.
  - Stewardship note: The phrase may be right, but it should be checked against the feeling of one continuous encounter rather than a repeated app action.

- Reconsider “Minu küsimus”.
  - Status: Needs decision.
  - Stewardship note: The wording should breathe naturally and avoid making the encounter feel like form submission or data entry.

- Protect silence, brevity and non-teaching.
  - Status: Protected quality.
  - Stewardship note: Silence is not absence. Brevity is not lesser care. Recognition is more important than teaching.

### Public Space and Reduction

- Remove duplicate CTA buttons from the homepage center.
  - Status: Needs verification before implementation.
  - Stewardship note: If the page already has clear invitation through header or footer, repeated central CTAs may create pressure instead of invitation.

- Keep homepage footer minimal.
  - Status: Needs decision.
  - Stewardship note: The homepage should feel open and quiet, not like a navigation dashboard.

- Avoid duplicating header links in footer.
  - Status: Needs decision.
  - Stewardship note: Duplication can make the public space feel crowded and app-like.

- Footer should not contain authenticated header items like Kohtumine, Teekond or Mälu.
  - Status: Needs decision.
  - Stewardship note: Public footer navigation should remain public and quiet; authenticated navigation belongs behind authenticated state.

- Public footer should likely have at most three quiet links.
  - Status: Needs decision.
  - Stewardship note: A small public set may preserve movement through the public world without turning the footer into a sitemap.

- Reconsider “Tutvu Expectumiga”.
  - Status: Needs decision.
  - Stewardship note: The phrase may be too explanatory or promotional depending on placement. It should be checked against invitation and quietness.

- Remove unnecessary “Algusesse” from `/return` if header/footer already provide navigation.
  - Status: Needs decision.
  - Stewardship note: `/return` should feel like a quiet pause, not a page that over-instructs movement.

- Signed-in homepage should not show unnecessary public CTA buttons.
  - Status: Needs verification and decision.
  - Stewardship note: Authenticated state should not create contradictory invitations to begin what is already accessible.

### Language and Naming

- Clarify Aim and Kaja usage in Estonian.
  - Status: Needs decision.
  - Stewardship note: The language should preserve the distinction between Aim as presence and kaja as echo without becoming explanatory in a heavy way.

- Avoid “Kustuta Kaja” if it sounds like deleting a person or name.
  - Status: Needs decision.
  - Stewardship note: If Kaja reads as a name, deletion language may carry the wrong feeling.

- Use lowercase kaja where it means echo, not name.
  - Status: Needs decision.
  - Stewardship note: This may help distinguish concept, trace and name without adding explanation.

- Reconsider “osaline” where it does not breathe naturally.
  - Status: Needs decision.
  - Stewardship note: The word may be accurate but could feel formal or administrative in some contexts.

- Preserve “Mis vaibub või jääb tahaplaanile”.
  - Status: Protected wording.
  - Stewardship note: This phrase holds reduction without violence; it lets something recede without being erased.

- Preserve “Teema on kordumised kohtumiste jälgedes”.
  - Status: Protected wording.
  - Stewardship note: This phrase keeps Theme grounded in traces rather than imposed interpretation.

- Keep language non-teaching, non-coaching and non-diagnostic.
  - Status: Protected quality.
  - Stewardship note: Expectum language should help something become visible, not instruct a person what it means.

### Public Content Structure

- `/expectum-language` has a naming inconsistency: Keel vs Vormiraamat.
  - Status: Needs verification and decision.
  - Stewardship note: The public language path should not split its own name unless the distinction is intentional and clear.

- Symbols and language may need one consolidated public path.
  - Status: Needs decision.
  - Stewardship note: If both explain the form of Expectum, they may belong closer together; consolidation should not be done before the public-space feeling is verified.

- Aim, Aim memory and Human-and-AI may need one consolidated public path.
  - Status: Needs decision.
  - Stewardship note: These pages may currently describe related aspects of Aim and human-AI relation. Consolidation could reduce repetition if it preserves clarity.

- Avoid more than three public footer links where possible.
  - Status: Needs decision.
  - Stewardship note: Footer navigation should support quiet movement, not explain the whole system.

### Visual Stewardship

- Expectum header should likely be word-only, breathing, without symbol.
  - Status: Needs decision.
  - Stewardship note: The wordmark may carry the space more quietly when it is not visually competing with symbols used for concepts.

- Symbol optical sizes should be consistent across pages.
  - Status: Needs verification.
  - Stewardship note: Symbol meanings have been protected; the remaining issue is optical balance and consistency.

- Background tone should be consistent across pages.
  - Status: Needs verification.
  - Stewardship note: Background inconsistency can make pages feel like different applications rather than one space.

- Card backgrounds should feel coherent.
  - Status: Needs verification.
  - Stewardship note: Card consistency work has begun, but final design finish should check whether card treatments now breathe together.

- Homepage text may need quiet line-by-line appearance.
  - Status: Needs decision.
  - Stewardship note: The homepage is an invitation. Its rhythm should support glimpse and possible direction without feeling like marketing text.

- Avoid visual noise on 14-inch laptop screens.
  - Status: Needs verification.
  - Stewardship note: The system should remain spacious on common laptop sizes, not only large desktop screens.

### Navigation and Route Stewardship

- Teekond footer should include Suund `/trajectory` rather than Mälu.
  - Status: Needs decision.
  - Stewardship note: Teekond and Suund are more directly related than Teekond and Mälu in this navigation context.

- Legacy routes `/movement-map` and `/theme-evolution` still need decision.
  - Status: Needs decision.
  - Stewardship note: Previous navigation work identified them as legacy or hidden. They should not be removed until the route decision is explicit.

- `/profile` should remain quiet or redirect unless it clearly serves the room.
  - Status: Needs decision.
  - Stewardship note: `/profile` should not become a dashboard. If it remains, it should act as quiet orientation only.

### Memory and Continuity

- Path and Statistics still partly mix local and remote data.
  - Status: Needs verification.
  - Stewardship note: Memory normalization reduced inflated repetition, but local/remote divergence remains a known area for future care.

- Memory should support continuity without active burden.
  - Status: Protected quality.
  - Stewardship note: Memory is remembering, not holding on. Archive or reduction of active use is preferred over destruction.

- Avoid misleading “clear memory” behavior unless fully true.
  - Status: Needs decision.
  - Stewardship note: If traces remain part of encounter or journey, the language should not imply complete deletion unless the system truly performs it.

## Decision Gates

The following items need explicit decision before implementation:

- Whether the public footer should have a fixed maximum of three links, and which links belong there.
- Whether homepage central CTA buttons should be reduced, and how signed-in homepage state should behave.
- Whether the Expectum header should become word-only without a symbol.
- Whether “Tutvu Expectumiga”, “Jätka kohtumist”, “Minu küsimus”, “osaline” and “Kustuta Kaja” should be changed.
- Whether lowercase `kaja` should be used where echo is meant rather than a name.
- Whether `/expectum-language` should be named Keel, Vormiraamat or something else.
- Whether Symbols and Language should become one consolidated public path.
- Whether Aim, Aim Memory and Human-and-AI should become one consolidated public path.
- Whether `/movement-map` and `/theme-evolution` should be removed, redirected or preserved as hidden legacy routes.
- Whether `/profile` should remain a quiet orientation page, redirect elsewhere or be removed from user-facing consideration.
- Whether Teekond footer navigation should replace Mälu with Suund `/trajectory`.
- What memory-clearing language should become if full deletion is not the intended long-term model.
- When and how One Meeting should move from decision into implementation.

## First Recommended Implementation Candidate

The smallest safe first PR after this backlog is a Public/Homepage Reduction Pass.

Suggested scope:

- Verify the current homepage public and signed-in states.
- Remove duplicate central CTA buttons only where navigation remains clear.
- Keep the homepage footer minimal.
- Avoid duplicating header links in the footer.
- Do not change routes, prompts, Supabase logic, auth behavior, memory behavior or visual language.

Why this first:

- It follows the Reduction Principle.
- It is likely smaller than One Meeting implementation.
- It protects public invitation before deeper structural changes.
- It can be done without touching prompts, data, auth, memory or routes.

Status: Needs verification before implementation.
